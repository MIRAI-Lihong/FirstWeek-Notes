# 运算符的扩展

## 指数运算符（`**`）

```js
// 2 ** 2 // 4
// 2 ** 3 // 8

// 相当于 2 ** (3 ** 2)
// 2 ** 3 ** 2
```

可以和等号结合（`**=`）

```js
a **= 2 //等同于 a = a * a;
```

## 链判断运算符（`?.`）

```js
// 链判断运算符左侧的对象是否为null或undefined。如果是的，就不再往下运算，而是返回undefined
const firstName = message?.body?.user?.firstName || 'default'

// 如果方法存在才执行
iterator.return?.()
```

> 注意点
>
> 短路机制：只要不满足条件，不会向下执行
> 一般来说，使用?.运算符的场合，不应该使用圆括号
> 右侧不得为十进制数值

报错场景

```js
// 构造函数
new a?.()
new a?.b()

// 链判断运算符的右侧有模板字符串
a?.`{b}`
a?.b`{c}`

// 链判断运算符的左侧是 super
super?.()
super?.foo

// 链运算符用于赋值运算符左侧
a?.b = c
```

## Null 判断运算符

```js
// 默认值只有在左侧属性值为null或undefined时，可以和 ?. 配合使用
const animationDuration = response.settings?.animationDuration ?? 300
```

## 逻辑赋值运算符

基本语法：

```js
// 或赋值运算符
x ||= y
// 等同于
x || (x = y)

// 与赋值运算符
x &&= y
// 等同于
x && (x = y)

// Null 赋值运算符
x ??= y
// 等同于
x ?? (x = y)
```

使用场景：

```js
// 老的写法
user.id = user.id || 1

// 新的写法
user.id ||= 1

// 为不存在的属性设置默认值
function example1(opts) {
  opts.foo = opts.foo ?? 'bar'
  opts.baz ?? (opts.baz = 'qux')
}

function example2(opts) {
  opts.foo ??= 'bar'
  opts.baz ??= 'qux'
}
```

## #!命令

```js
// 写在脚本文件第一行
#!/usr/bin/env node
'use strict';
console.log(1);

// 写在模块文件第一行
#!/usr/bin/env node
export {};
console.log(1);
```

```bash
# 以前执行脚本的方式
$ node hello.js

# hashbang 的方式
$ ./hello.js
```
