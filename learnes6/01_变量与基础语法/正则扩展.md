# 正则的扩展

## RegExp 构造函数

ES5 中，如果第一个参数是正则对象，不允许传第二个参数（修饰符），否则报错。

ES6 改变了这一行为，允许第二个参数覆盖原有修饰符：

```js
new RegExp(/abc/gi, 'i').flags // "i"，原来的 ig 被覆盖为 i
```

## 新增修饰符

### u 修饰符

处理大于 `\uFFFF` 的 Unicode 字符（四字节 UTF-16 编码）时，需要加 `u` 修饰符才能正确识别：

```js
/^\uD83D/u.test('\uD83D\uDC2A') // false
/^\uD83D/.test('\uD83D\uDC2A')  // true
```

加了 `u` 后影响以下行为：

- 点字符 `.` 能匹配四字节 Unicode 字符
- `\u{61}` 大括号 Unicode 写法被正确识别
- 量词、预定义模式能正确匹配四字节字符
- `i` 修饰符能识别非规范 Unicode 字母

```js
/\u{61}/u.test('a')    // true
/^\S$/u.test('𠮷')    // true
```

### y 修饰符

和 `g` 类似都是全局匹配，区别在于 `y` 要求每次匹配必须从剩余字符串的**头部**开始：

```js
var s = 'aaa_aa_a'
var r1 = /a+/g
var r2 = /a+/y

r1.exec(s) // ["aaa"]
r2.exec(s) // ["aaa"]

r1.exec(s) // ["aa"]  g 修饰符跳过了 _
r2.exec(s) // null    y 修饰符要求从头匹配，_ 不是 a，失败
```

`y` 修饰符的一个应用，是从字符串提取 `token`

```js
const TOKEN_Y = /\s*(\+|[0-9]+)\s*/y
const TOKEN_G = /\s*(\+|[0-9]+)\s*/g

tokenize(TOKEN_Y, '3 + 4')
// [ '3', '+', '4' ]
tokenize(TOKEN_G, '3 + 4')
// [ '3', '+', '4' ]

function tokenize(TOKEN_REGEX, str) {
  let result = []
  let match
  while ((match = TOKEN_REGEX.exec(str))) {
    result.push(match[1])
  }
  return result
}
```

### s 修饰符

默认情况下，`.` 无法匹配行终止符（`\n`、`\r` 等），`s` 修饰符让 `.` 匹配任意字符：

可通过 `dotAll` 属性判断是否启用：

```js
const re = /foo.bar/s
// 另一种写法
// const re = new RegExp('foo.bar', 's');

re.test('foo\nbar') // true
re.dotAll // true
re.flags // 's'
```

## 新增属性

| 属性      | 说明                  |
| --------- | --------------------- |
| `flags`   | 返回正则所有修饰符    |
| `unicode` | 是否设置了 `u` 修饰符 |
| `sticky`  | 是否设置了 `y` 修饰符 |

## 具名组匹配

```js
const re = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/
const result = re.exec('2024-06-20')

result.groups.year // '2024'
result.groups.month // '06'
result.groups.day // '20'
```

## String.prototype.matchAll

对一个正则（要有 `g` 修饰符）返回所有匹配的遍历器

```js
const string = 'test1test2test3'
const regex = /t(e)(st(\d?))/g

for (const match of string.matchAll(regex)) {
  console.log(match)
}
// ["test1", "e", "st1", "1", ...]
// ["test2", "e", "st2", "2", ...]
// ["test3", "e", "st3", "3", ...]
```
